# Huber Delta Sensitivity in 3D Trajectory Reconstruction

## Research Question
How does the Huber loss sensitivity parameter `delta` control the robustness-accuracy trade-off under outlier corruption for synthetic 3D trajectory reconstruction?

This project implements a controlled PyTorch experiment comparing `MSE` and `Huber` losses for:
1. Delta sensitivity (`delta in [0.1, 0.5, 1.0]`)
2. Robustness vs accuracy trade-offs under outliers
3. Depth sensitivity (2, 3, 5 hidden-layer MLPs)
4. Computational efficiency
5. Statistical significance across repeated random seeds (seed-level paired tests)

## Synthetic Data Generation
Trajectory definition:
- `x = cos(t)`
- `y = sin(t)`
- `z = 0.2 * t`

Data pipeline:
1. Create clean trajectory for `t in [0, t_max]`.
2. Normalize time to `[0, 1]` for model input.
3. Add small Gaussian noise to observations.
4. Inject outliers either:
- at fixed indices (default: `[30, 80, 130, 170]`), or
- by random positions from configurable outlier ratio.

## Model and Losses
Model:
- MLP regressor with Tanh activations
- Hidden width default `128`
- Depth sweep: `2`, `3`, `5` hidden layers
- Shared Xavier initialization for fair comparison

Loss:
1. Main regression loss:
- `MSE`
- `Huber(delta)`
2. Auxiliary endpoint loss:
- forces first and last predicted points toward clean endpoints
- weight default `10.0`
3. Auxiliary smoothness loss:
- mean squared second-order differences
- weight default `1.0`

Total loss:
`L_total = L_regression + w_endpoint * L_endpoint + w_smooth * L_smoothness`

## Training Setup
- Optimizer: Adam
- Learning rate: `0.003`
- Weight decay: `1e-5`
- Default epochs: `8000`
- Optional early stopping available (disabled by default)
- Device auto selection: CPU/GPU

## Statistical Testing Rule (Important)
Statistical tests are run on final seed-level metrics only (never per-epoch losses):
- Use paired t-test when normality of paired differences is acceptable (Shapiro check).
- Otherwise use Wilcoxon signed-rank test.
- All test outputs are saved to CSV.

```bash
pip install -r requirements.txt
```

## Run Experiments
From project root:

Full required experiment groups (A-E):
```bash
python -m src.main --config configs/default.yaml
```

Only delta sensitivity:
```bash
python -m src.main --config configs/delta_sweep.yaml
```

Only depth sensitivity:
```bash
python -m src.main --config configs/depth_sweep.yaml
```

Only outlier severity study:
```bash
python -m src.main --config configs/outlier_study.yaml
```

Optional overrides:
```bash
python -m src.main --config configs/default.yaml --epochs 2000 --seeds 0,1,2
python -m src.main --config configs/default.yaml --save_checkpoints
python -m src.main --config configs/default.yaml --output_dir custom_results
```

## Output Files
All outputs go to `results/` (or `--output_dir` if provided):

CSV outputs in `results/csv/`:
- `run_metrics.csv`: raw run-level metrics (one row per seed/config run)
- `summary_metrics.csv`: mean/std across seeds
- `statistical_tests.csv`: paired test results
- `report_summary_compact.csv`: compact report-ready summary (`mean ± std`)
- `figure_index.csv`: auto-generated figure catalog

Figures in `results/figures/`:
1. 3D trajectory comparison (clean, noisy, outliers, prediction)
2. 2D projections (XY, YZ, XZ)
3. Training loss curves
4. Delta sensitivity plot
5. Efficiency plot
6. Summary bar plots with seed-level error bars

Reproducibility configs in `results/configs/`:
- `resolved_config.json`
- one per-run JSON config snapshot

Optional checkpoints in `results/checkpoints/`:
- generated only when checkpoint saving is enabled
